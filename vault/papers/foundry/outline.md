# Outline

## One-Sentence Version

Foundry compiles curated workflow-construction knowledge into target-specific agent skills, preserving provenance between human-inspectable sources and executable agent procedures.

## Reader

Bioinformatics readers interested in workflow interoperability, agent-assisted workflow authoring, and provenance of generated scientific artifacts.

## Proposed Structure

1. Problem: agent-authored workflows fail in predictable, detectable ways.
2. Diagnosis: monolithic skills and runtime RAG do not provide enough inspectability or completeness for schema-bound workflow authoring.
3. Architecture: Patterns, Molds, Pipelines, Schemas, Casts, provenance.
4. Validation loop: gxwf as deterministic inner-loop feedback.
5. Case study: Nextflow or CWL pipeline summarized and partially converted through cast skills.
6. Comparison: MCP/resources, agent skills, llms.txt, Corpus2Skill, vector RAG, OpenAPI generators.
7. Discussion: compile-time grounding with provenance for high-stakes computational biology.

## Positioning

Lead with the knowledge-to-skill compiler idea, not the website. The site is the human view of the same content.
